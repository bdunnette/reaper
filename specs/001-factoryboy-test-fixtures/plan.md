# Implementation Plan: Replace Static Test Fixtures with factory_boy

**Branch**: `001-factoryboy-test-fixtures` | **Date**: 2026-06-11 | **Spec**: [spec.md](file:///C:/Users/dunn0172/Documents/GitHub/reaper/specs/001-factoryboy-test-fixtures/spec.md)

## Summary

The feature replaces the static, hardcoded dictionary mock objects in `tests/test_client.py` with dynamic, customizable mocks using `factory_boy`. This will allow more flexible testing of autocomplete, search, details, and pagination flows.

## Technical Context

**Language/Version**: Python >= 3.14 (per Reaper Constitution)

**Primary Dependencies**: `factory_boy` (for mock factories), `httpx` (mocked in tests), `pydantic` (validating reaper models)

**Storage**: N/A

**Testing**: `pytest` and `pytest-asyncio`

**Target Platform**: Any standard OS supported by python (windows, linux, macos)

**Project Type**: library

**Performance Goals**: N/A (Test execution performance should remain instantaneous)

**Constraints**: Must run with existing mock client integration tests; no changes to production code.

**Scale/Scope**: ~10 test functions in `tests/test_client.py` to be modernized.

## Constitution Check

*GATE: Must pass before Phase 0 research. Re-check after Phase 1 design.*

- **Principle I: Browser Fingerprint Emulation & Proxy Integration**: Pass. Factories do not affect requests header emulation, but support it.
- **Principle II: Pydantic-Validated Models**: Pass. The factories generate mock JSON structures that will be successfully validated by the Reaper Pydantic models.

- **Principle IV: Zero-Dependency Dataframe Interoperability**: Pass. Mocks generated by factories will be processed by narwhals tests.
